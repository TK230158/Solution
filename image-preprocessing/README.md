# 画像前処理 検証デモ

> **注記**: これは鉄骨文字認識アプリ全体の中で、**画像処理班・前処理担当（坂井）の実装範囲**を単独で検証するためのデモアプリです。  
> 最終成果物（鉄骨文字認識アプリ本体）とは別物です。

---

## システム全体における位置づけ

鉄骨文字認識アプリは複数の処理ステップで構成されており、このデモは **Step 1「画像前処理」** の部分を担当しています。

```
【鉄骨文字認識アプリ 全体フロー】

  撮影
    │
    ▼
┌─────────────────────────────┐
│  Step 1: 画像前処理           │ ← このデモの担当範囲
│  （照明補正・ノイズ除去・二値化）  │
└─────────────────────────────┘
    │  前処理済み画像
    ▼
┌─────────────────────────────┐
│  Step 2: 文字領域検出         │ ← 他担当
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Step 3: 文字認識（OCR）      │ ← 他担当
└─────────────────────────────┘
    │
    ▼
  認識結果の表示・出力
```

このデモでは、**前処理が正しく機能しているか・処理時間が要件を満たすか** を単独で確認するために作成しました。

## 前処理の処理内容

撮影した鉄骨画像に対して以下の 5 ステップを適用し、文字認識エンジンに渡しやすい形に変換します。

```
元画像（カラー）
    │
    ▼
① リサイズ         最長辺を 960px に縮小（アスペクト比維持）
    │
    ▼
② グレースケール化  BGR → Gray（1チャンネル化）
    │
    ▼
③ CLAHE            局所的なコントラスト補正（照明ムラ・影の対策）
    │
    ▼
④ ガウシアンブラー  ノイズ除去（カーネルサイズ 5×5）
    │
    ▼
⑤ 適応的二値化      局所平均に基づく閾値処理（文字の白黒化）
    │
    ▼
前処理済み画像（白黒2値）→ 文字認識ステップへ渡す
```

### パラメータ

| ステップ | パラメータ | 値 |
|---------|-----------|---|
| リサイズ | 最大サイズ | 960 px |
| CLAHE | clipLimit | 2.0 |
| CLAHE | tileGridSize | 8×8 |
| ガウシアンブラー | カーネルサイズ | 5×5 |
| 適応的二値化 | blockSize | 11 |
| 適応的二値化 | 定数 C | 2.0 |

## デモアプリの使い方

1. アプリを起動すると、テスト用の鉄骨画像 18 枚が自動で読み込まれる
2. 上部の **「前処理モード」** スピナーで処理モードを選択
3. ドロップダウンで画像を選択（◀ / ▶ でも切り替え可能）
4. **「前処理実行」** ボタンをタップ → 元画像と前処理後の画像を並べて表示
5. **「詳細計測」** ボタンで各ステップごとの処理時間を確認可能
6. **「全画像 一括処理」** ですべての画像を連続処理してサマリー表示
7. **「📥 保存」** / **「全画像 一括保存」** で前処理済み画像をギャラリーに保存

## 起動方法

### A. Android エミュレータで起動する

> Android Studio が必要です（[ダウンロード](https://developer.android.com/studio)）

1. **Android Studio を開く**  
   `File → Open` で `image-preprocessing/` フォルダを選択

2. **エミュレータを作成する**  
   `Tools → Device Manager → + Create Virtual Device`  
   - ハードウェア: **Pixel 7**（推奨）  
   - システムイメージ: **API 34（Android 14）**  
   - 「Finish」で作成  

3. **エミュレータを起動する**  
   Device Manager から ▶ ボタンでエミュレータを起動

4. **アプリをビルド＆起動する**  
   画面右上の **▶ Run ボタン** を押す（または `Shift + F10`）  
   → エミュレータにインストールされて自動で起動

5. **コマンドラインからインストールする場合**（Android Studio 不要）

   ```powershell
   cd image-preprocessing
   .\gradlew.bat installDebug
   ```

---

### B. Android 実機（スマートフォン）で起動する

> USB ケーブルでPCと接続する必要があります

#### 1. スマートフォン側の設定

1. **「設定」→「デバイス情報」→「ビルド番号」** を 7 回タップ  
   → 「開発者向けオプションが有効になりました」と表示される

2. **「設定」→「システム」→「開発者向けオプション」** を開く  
   （機種によって場所が異なる場合あり）

3. **「USBデバッグ」** をオン にする

#### 2. PC側の操作

1. USB ケーブルでスマートフォンとPCを接続する

2. スマートフォンに「**このコンピュータを信頼しますか？**」と表示されたら  
   → **「許可」** をタップ

3. デバイスが認識されているか確認する  
   ```powershell
   adb devices
   ```
   `List of devices attached` の下に `xxxxxxxx device` と表示されれば OK

#### 3. アプリのインストールと起動

**Android Studio から起動する場合:**
1. 画面上部のデバイス選択欄で接続したスマートフォンを選択
2. **▶ Run ボタン** を押す → スマートフォンに自動インストール＆起動

**コマンドラインから起動する場合:**
```powershell
cd image-preprocessing
.\gradlew.bat installDebug
adb shell am start -n com.example.imagepreprocessingtest/.MainActivity
```

#### 4. よくあるトラブル

| 症状 | 対処法 |
|------|--------|
| `adb devices` に表示されない | ケーブルを抜き差ししてスマートフォンで「接続方法」を「ファイル転送(MTP)」に変更 |
| `unauthorized` と表示される | スマートフォン画面で「USBデバッグを許可しますか？」を再確認して「許可」 |
| アプリが起動しない | `adb logcat` でログを確認する |
| OpenCV 初期化失敗 | armeabi-v7a / arm64-v8a ABI がサポートされているか確認（ほとんどの機種は問題なし） |

## ファイル構成

```
image-preprocessing/
├── app/src/main/
│   ├── java/com/example/imagepreprocessingtest/
│   │   ├── ImagePreprocessor.kt   ← 前処理ロジック（Step 1 の実装本体）
│   │   └── MainActivity.kt        ← デモ用 UI
│   └── assets/images/             ← テスト用鉄骨画像 18 枚

```

`ImagePreprocessor.kt` が Step 1 の実装本体です。最終的な文字認識アプリに組み込む際は、このクラスをそのまま利用します。
