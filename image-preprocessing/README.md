# 画像前処理 検証デモ

> **注記**: これは鉄骨文字認識アプリ全体の中で、**画像処理班・前処理担当（坂井）の実装範囲**を単独で検証するためのデモアプリです。  
> 最終成果物（鉄骨文字認識アプリ本体）とは別物です。

---

## システム全体における位置づけ

鉄骨文字認識アプリは複数の処理ステップで構成されており、このデモは **Step 1「画像前処理」** の部分を担当しています。

```
【鉄骨文字認識アプリ 全体フロー】

  撮影
    │
    ▼
┌─────────────────────────────┐
│  Step 1: 画像前処理           │ ← このデモの担当範囲
│  （照明補正・ノイズ除去・二値化）  │
└─────────────────────────────┘
    │  前処理済み画像
    ▼
┌─────────────────────────────┐
│  Step 2: 文字領域検出         │ ← 他担当
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Step 3: 文字認識（OCR）      │ ← 他担当
└─────────────────────────────┘
    │
    ▼
  認識結果の表示・出力
```

このデモでは、**前処理が正しく機能しているか・処理時間が要件を満たすか** を単独で確認するために作成しました。

---

## 前処理の処理内容

撮影した鉄骨画像に対して以下の 5 ステップを適用し、文字認識エンジンに渡しやすい形に変換します。

```
元画像（カラー）
    │
    ▼
① リサイズ         最長辺を 960px に縮小（アスペクト比維持）
    │
    ▼
② グレースケール化  BGR → Gray（1チャンネル化）
    │
    ▼
③ CLAHE            局所的なコントラスト補正（照明ムラ・影の対策）
    │
    ▼
④ ガウシアンブラー  ノイズ除去（カーネルサイズ 5×5）
    │
    ▼
⑤ 適応的二値化      局所平均に基づく閾値処理（文字の白黒化）
    │
    ▼
前処理済み画像（白黒2値）→ 文字認識ステップへ渡す
```

### パラメータ

| ステップ | パラメータ | 値 |
|---------|-----------|---|
| リサイズ | 最大サイズ | 960 px |
| CLAHE | clipLimit | 2.0 |
| CLAHE | tileGridSize | 8×8 |
| ガウシアンブラー | カーネルサイズ | 5×5 |
| 適応的二値化 | blockSize | 11 |
| 適応的二値化 | 定数 C | 2.0 |

| OpenCV 初期化 | ✅ 成功 |

## デモアプリの使い方

1. アプリを起動すると、テスト用の鉄骨画像 18 枚が自動で読み込まれる
2. ドロップダウンで画像を選択（◀ / 次▶ でも切り替え可能）
3. **「前処理実行」** ボタンをタップ → 元画像と前処理後の画像を並べて表示
4. **「詳細計測」** ボタンで各ステップごとの処理時間を確認可能

## ファイル構成

```
image-preprocessing/
├── app/src/main/
│   ├── java/com/example/imagepreprocessingtest/
│   │   ├── ImagePreprocessor.kt   ← 前処理ロジック（Step 1 の実装本体）
│   │   └── MainActivity.kt        ← デモ用 UI
│   └── assets/images/             ← テスト用鉄骨画像 18 枚
└── opencv/                        ← OpenCV 4.11 ライブラリ
```

`ImagePreprocessor.kt` が Step 1 の実装本体です。最終的な文字認識アプリに組み込む際は、このクラスをそのまま利用します。
